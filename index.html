Undergraduate Research Assistant

University of Waterloo
2024.3-2024.5

Network Research, Linux NIC Schedulers, Mininet, UDP, Python

Under the supervision of Prof. Mina Arashloo, I experimented to test the robustness of network traffic performance. I used network traffic simulations to test the fairness of Linux traffic control schedulers, QDiscs (Queuing Disciplines), with a focus on the HHF variant (Heavy-Hitter Filter).
I tested different QDisc configurations' impact on traffic shaping and throughput, aiming to understand better the behaviour of such QDisc variants and potentially disrupt a scheduler's fairness scheme. To achieve this, I designed the traffic flow generation algorithms and wrote the code to set up the experiment.
The multi-thread program created multiple continuous traffic flows, sending UDP packets under both light and heavy traffic conditions. For the same QDisc configuration, I compared the throughput plots for various traffic flow combinations. For different QDisc configurations, I compared the average throughput for all flows with the same traffic generator.
By comparing these generated plots, I noticed some exception cases with unexpected flow behaviour under some specific configurations. Although my research has concluded, the reasons for these exceptions remain areas for further investigation.

Systems Software Development Student

Blackberry, Ltd.
2023.9-2023.12

QNX, Cryptography, Virtual Box, Linux, System-level C, Python, JSON

I designed a Python test framework for Blackberry Certicom's system-level cryptography library, to support the application of a safety certification. The test framework aims to achieve (1) higher code coverage and (2) proof of zero false-negative test cases.
I generated JSON configuration files from the header files containing information for all individual functions. I then customized the configuration files to include setup and teardown processes for each function, as they are different steps in the encryption/decryption process. I integrated this framework into the testware Makefile system, allowing Python to generate the test files in C during the compilation process. The testware then executes these new test files and the manually written test cases.
I designed a runtime fault injection feature for the second goal using a statistical approach. The user inputs a number for the fault injection percentage, and using macros and an existing random number generator, the framework decides whether to insert a fault to a test case based on that percentage. With no false-negative test cases, the detected fault percentage approached the same percentage as the user input in the long run.
The automated test framework supports testing all ECC variant algorithms and SHA256. The development is under Linux environment, using Virtual Box.
