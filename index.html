Undergraduate Research Assistant

University of Waterloo

• Explored Linux Qdisc scheduler traits and designed customized traffic flow generation algorithms using Python.

Under the supervision of Prof. Mina Arashloo, I conducted an experiment to test the robustness of network traffic performance. I used network traffic simulations to test the fairness of Linux traffic control schedulers, QDiscs (Queuing Disciplines), with a focus on the HHF varient (Heavy-Hitter Filter).
I tested different QDisc configuration's impact on traffic shaping and throughput, aiming to better understand the behaviour of such QDisc varient and potentially disrupt a scheduler's fairness scheme. To achieve this, I designed the traffic flow generation algorithms, and wrote the code to setup the experiment.
The multi-thread program created multiple continuous traffic flows, sending packets under both light and heavy traffic conditions. For the same QDisc configuation, I compared the throughput plots for various traffic flow combinations. For different QDisc configurations, I compared the average throughput for all flows with the same traffic generator.
By comparing these generated plots, I noticed some exception cases with unexpected flow behaviour under some specific configurations. Although my research has concluded, the reasons for these exception remain areas for further investigation.

Systems Software Development Student

Blackberry, Ltd.

2023.9-2023.12

• Developed a Python test framework for QNX's system-level cryptography library on Linux, supporting safety certification application process. Included ECC and SHA256 algorithm tests.
• Improved test coverage with code generation. Integrated framework into test ware compilation using Makefile.
• Implemented runtime fault injection for error detection using statistical methods, ensuing its accuracy and reliability.

I designed a Python test framework for Blackberry Certicom's system-level cryptography library, to support the application of a safety certification. The goal of the test framework is to achieve (1) higher code coverage and (2) proof of zero false-negative test cases.
From the header files, I generated JSON configuration files containing information for all individual functions. I then customized the configuration files to include setup and teardown processes for each functions, as they are different steps in the entire encryption/decription process. I integrated this framework into the testware Makefile system, allowing Python to generate the test files in C during compilation process. These new test files will be executed along with the manually written testware.
For the second goal, I designed a runtime fault injection feature using a statistical approach. The user inputs a number for the fault injection percentage, and using macros and an existing random number generator, the framework decides whether to insert a fault to a test case based on that percentage. With no false-negative test cases, the detected fault percentage approached the exact same percentage as the user input in a long run.
The automated test framework supports the testing of all ECC varient algorithms and SHA256. The development is in Linux environment, using Virtual Box.
